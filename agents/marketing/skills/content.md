# Content Skill (owned by Marketing)

## Purpose

Produce long-form content — blog posts, guides, thought-leadership pieces, ebooks, talk scripts — that's grounded in real customer/account context and consistent with the company's outreach messaging. The output is a publishable draft, not a one-shot brainstorm.

## Inputs

- `context/product-overview.md`
- `context/customers.md`
- `context/competitive-landscape.md` (only the ICP section relevant to the topic)
- `output/icp.md`
- `output/outreach-strategy.md` (for messaging consistency)
- The topic, format, and audience (from the user via the Orchestrator)
- On demand: `output/research/{icp}/{slug}.md`, `output/cs/{customer}/`, `output/outreach-learnings.md`

## Output

Write to `output/marketing/content/{topic-or-slug}-{YYYY-MM-DD}.md`.

---

## Methodology

### Step 1: Lock the angle

Before writing, lock three things in plain text:

1. **Audience:** which ICP / persona is this for? (e.g., "Heads of T&S at marketplaces > 500 employees")
2. **Job to be done:** what does the reader want to do after reading? (subscribe, book a demo, share with team, change a decision)
3. **Singular thesis:** the one sentence the entire piece argues. If you can't write it in one sentence, you don't have a thesis yet.

The angle should echo the angles Sales is using in `output/outreach-strategy.md` — content and outbound should feel like the same company.

### Step 2: Outline before drafting

Produce a section outline:

- Hook (1-2 sentences) — the surprising or specific opening that earns the read
- Thesis (1 sentence)
- 3-5 sub-points, each with:
  - The claim
  - One concrete example (real customer / real news / real research file / real number)
  - The "so what" — what the reader should do or believe differently
- Conclusion — restate the thesis with the strongest example, end with a CTA

Show the outline to the user. Confirm before drafting.

### Step 3: Draft

Write the full piece against the outline. Rules:

- **Specific over generic.** "63% of marketplace abuse goes undetected for 30+ days" beats "abuse is a big problem." If you don't have a specific number, find one or cut the claim.
- **Show the work.** Cite sources inline (link to news, to research file, to customer story). Never fabricate a stat to make a point.
- **One idea per paragraph.** No throat-clearing.
- **Plain language.** Cut jargon unless it's the audience's literal vocabulary.
- **Consistent voice.** Match prior pieces in `output/marketing/content/`. Read one for tone reference if uncertain.

### Step 4: Confirm and write

Show the full draft to the user in chat. Offer:
- Save as-is
- Edit specific sections (loop)
- Discard

Only write the file on confirmation.

## Quality Checklist

- [ ] Single thesis sentence at the top — readable in 5 seconds
- [ ] Every sub-point has a real example (real customer / real number / real news), not a manufactured one
- [ ] No fabricated stats, customer quotes, or product capabilities
- [ ] Voice matches prior content in `output/marketing/content/`
- [ ] Audience and job-to-be-done explicit in the file header
- [ ] User saw and confirmed the draft before the file was written
- [ ] CTA is specific and named (not "learn more")
